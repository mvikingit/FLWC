# FLWC Chief Engineer Blueprint

Node: P620_FLWC_CE_CCP_STATE_PACK_CODEX_A_IMPLEMENTATION_V1

FLWC is an independent Financial LLM Wiki Compiler. It compiles authorized financial text knowledge into auditable, bitemporal, versioned wiki and candidate evidence artifacts.

Current phase: B0 fixture-only typed schema and deterministic validator package. B0 is limited to synthetic fixtures, typed schema structures, deterministic validators, local unit tests, source-authority archival, and this Chief Engineer state pack.

FLWC is not production. FLWC is not live trading. FLWC is not market-data authority. FLWC does not open real source ingestion, vendor APIs, paid source access, web scraping, DuckDB canonical seed, runtime services, external docking, scanner behavior, order intent, broker execution, or position sizing.

CE-CCP-FLWC operating model:

- Commander is the project owner and sensitive-surface authorizer.
- FLWC Chief Engineer owns node boundaries, state, non-claim preservation, and final verdicts.
- Codex A is the bounded implementation agent.
- Codex B is the read-only adversarial audit agent.
- GitHub, CI, push-gate output, GitHub App, and local LLM output are evidence channels only.
- A node closes only through Chief Engineer verdict backed by repo artifacts, tests, non-claims, and state-pack update.

The Chief Engineer state pack is the durable context anchor for FLWC-QEP work. It does not replace WS-FLWC constitutional law or A1-A5 source authority.
